# 🧩 Java Test Automation Design Patterns

A curated collection of **design patterns implemented in Java**, focused on improving **test automation architecture, maintainability, and scalability**.

This repository is built from an **SDET perspective**, demonstrating how common design patterns can be applied in real-world automation frameworks.

---

## 🚀 Patterns Covered

### 🔹 SOLID Principles

* **SRP (Single Responsibility Principle)**
  Ensures each class has one clear responsibility, improving readability and maintainability.

---

### 🔹 Behavioral Patterns

* **Strategy Pattern**
  Enables dynamic behavior selection (e.g., switching test strategies, drivers, or execution flows).

---

### 🔹 Creational Patterns

* **Factory Pattern**
  Centralized object creation (commonly used for WebDriver/API client initialization).

---

### 🔹 Functional / Java-Specific Patterns

* **Execute Around Pattern**
  Simplifies resource handling (setup/teardown logic), often used in test wrappers and utilities.

---

### 🔹 Custom / Practical Patterns

* **JFRog Pattern**
  A practical pattern used to structure reusable and modular automation components.

---

## 💡 Why This Repository?

In test automation, poorly structured code leads to:

*  High maintenance cost
*  Flaky tests
*  Tight coupling

This repo demonstrates how design patterns help:

*  Improve code reusability
*  Reduce duplication
*  Build scalable automation frameworks
